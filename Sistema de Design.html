Introdução

A API CPSGUI é uma API de interface gráfica para uso de estudantes iniciantes em programação e visa a ser fácil o suficiente para tarefas como criar uma janela e alguns componentes simples.

CPSGUI é uma sigla para Cross-Platform Simple Graphical User Interface, ou seja, Interface Gráfica de Usuário Simples MultiPlataforma. Isso porque a API oculta detalhes específicos do sistema operacional, podendo ser implementada em diferentes sistemas. Atualmente, ela tem implementações para Windows e Linux.

O funcionamento da API é baseado em eventos. Isso significa que haverá um laço de repetição que constantemente espera novos eventos e que os trata quando os recebe. Os eventos existentes na API são o evento de clique de botão e de pressionamento de tecla. Um evento de clique de botão é recebido quando o usuário clica em um botão.

Diferenças entre CLI e GUI

Um programa com interface de linha de comando (CLI - Command Line Interface), ou seja, baseado em texto e digitação com o teclado, possui diferenças fundamentais de interface de usuário em relação a um programa com interface gráfica de usuário (GUI - Graphical User Interface). Em uma interface baseada em texto, o programador possui controle sobre o que o usuário fará no momento. Se, por exemplo, o programador precisa do nome do usuário para prosseguir, basta exibir algo como “Digite seu nome:”, e o usuário poderá inserir o seu nome. Em uma interface gráfica de usuário, contudo, o usuário decide o que irá fazer e em que momento. Se há, por exemplo, um formulário com diversos campos, o usuário poderá preencher o segundo campo antes do primeiro ou vice-versa. Além disso, no caso de um formulário, é natural que sejam preenchidos todos os campos e então clicado em um botão ao final. Após esse clique, todas as informações provavelmente serão coletadas de uma vez só, diferentemente do que aconteceria em um programa baseado em texto.

Tendo em vista as particularidades de uma interface gráfica de usuário, fica clara a necessidade de uma forma diferente de programar a interação com o usuário. A forma utilizada na API CPSGUI é a programação baseada em eventos. Basicamente, isso consiste na constante leitura por ações do usuário e uma tomada de decisão de acordo com a ação realizada. Na programação de interfaces baseadas em texto é comum haver menus de opções, listando as ações possíveis ao usuário e solicitando um número ou letra de acordo com o menu desejado. Para isso, é utilizado um laço de repetição e uma estrutura de decisão. A ideia da programação baseada em eventos é virtualmente a mesma. O laço de repetição também serve para evitar que o programa simplesmente termine, pois sem um laço de repetição o programa chegaria ao ponto de retorno.

A conversão da interface de usuário de um programa baseado em texto para uma interface gráfica é mais simples ou mais complexa, dependendo de como o programa original foi estruturado. Programas com boa modularidade tendem a ser mais fáceis de converter. Se existem funções que implementam apenas as regras de negócio do programa e que não interagem diretamente com o usuário, a conversão é relativamente simples. Essencialmente, é necessário transformar as leituras de dados em campos de texto, as escritas de dados em campos de texto estático e as escolhas e/ou confirmações (apertar “Enter” para indicar o término da entrada seria um exemplo de confirmação) em botões. O programa deveria, então, constantemente obter eventos, que representam as ações do usuário, e executar o código relativo à ação realizada.

O código a seguir mostra um exemplo de obtenção de dados de campos de texto a partir do clique em um botão.

#include "build/API.h"
int main() {
    Window window = createWindow("Formulário", 0, 0, 510, 160, rgb(255, 255, 255));
    StaticText labelNome = createStaticText(window, 10, 10, 100, 20, 0, "Nome");
    TextField entradaNome = createTextField(window, 10, 30, 100, 30, rgb(107, 107, 107), "", 0);
    StaticText labelIdade = createStaticText(window, 10, 70, 100, 20, 0, "Idade");
    TextField entradaIdade = createTextField(window, 10, 90, 100, 30, rgb(107, 107, 107), "", 0);
    Button botaoConfirmar = createButton(window, 10, 130, 100, 20, rgb(107, 0, 100), "Confirmar", rgb(0, 0, 0), 0);

    while (1) {
        Event event = getEvent(window);
        if (event.type == CLICK_EVENT) {
            switch (event.data) {
            case 0:
            {
                char nome[50];
                char idade[5];
                getTextFieldText(entradaNome, nome);
                getTextFieldText(entradaIdade, idade);
            }
            break;
            default:
            break;
            }
        }
    }
    return 0;
}

No exemplo acima são criados vários componentes, incluindo um botão. O botão recebe o número 0 como identificador. No laço de eventos é tratado o evento de clique em botão, e quando o botão com identificador 0 é clicado os dados do campos de texto são guardados nas variáveis nome e idade.

Janela

A criação de janela é feita com a função createWindow e destruída com a função destroyWindow. O título da janela pode ser alterado usando a função setWindowTitle. Atualmente, a API suporta apenas uma única janela de tamanho fixo.

Funções de janela

  • createWindow
  • setWindowTitle
  • destroyWindow

O exemplo abaixo ilustra a utilização das funções acima. 

#include "build/API.h"
int main() {
    Window window = createWindow("Exemplo", 10, 10, 500, 400, rgb(255, 100, 100));
    setWindowTitle(window, "Novo título");
    Button button = createButton(window, 20, 20, 100, 20, rgb(255, 255, 255), "Fechar", rgb(0, 0, 0), 0);
    while (1) {
    Event event = getEvent(window);
    if (event.type == CLICK_EVENT) {
        switch (event.data) {
        case 0:
            destroyWindow(window);
            return 0;
        break;
        default:
        break;
        }
    }
    }
    return 0;
}

Nesse exemplo, é criada uma janela, seu título é alterado imediatamente após a criação, e a janela é destruída quando o usuário clica no botão com identificador 0 (botão chamado “Fechar”, nesse caso).

Eventos

Os eventos podem ser obtidos com a função getEvent. O evento consiste em uma estrutura de dado composta por um tipo (membro “type”) e um dado (membro “data”). O evento de clique de botão (CLICK_EVENT) tem no membro “data” o identificador do botão que foi clicado. Virtualmente toda a aplicação terá um laço de repetição que constantemente lê eventos, como no exemplo abaixo.

while (1) {
    Event event = getEvent(window);
    if (event.type == CLICK_EVENT) {
      switch (event.data) {
      case 0:
        {
          char nome[50];
          char idade[5];
          getTextFieldText(entradaNome, nome);
          getTextFieldText(entradaIdade, idade);
        }
        break;
      default:
        break;
      }
    }
}

Esse exemplo apenas obtém o texto de campos de texto, mas aplicações reais terão maior quantidade de lógica dentro do laço.

Componentes

A API possui os seguintes componentes: texto estático, campo de texto, botão, retângulo e linha.

O texto estático consiste em texto que pode ser exibido e alterado pelo programa, mas não é editável pelo usuário. Esse tipo de componente serve para informar o usuário em relação a resultados ou para descrever o propósito de algum componente adjascente. O propósito é similar à função printf, da biblioteca padrão da linguagem C, mas é importante lembrar que o componente de texto estático não é capaz de formatar dados, diferentemente da função printf.

O campo de texto consiste em uma caixa em que o usuário pode digitar texto. O programa tem a possibilidade de ler e alterar o conteúdo desse componente, mas é mais comum que apenas se leia o que o usuário digitou. Para que o usuário possa digitar algo no campo de texto, ele deve primeiro clicar sobre o componente. Atualmente, a API apenas suporta a inserção de caracteres e a remoção de caracteres com a tecla “Backspace”. Esse tipo de componente serve para que o usuário possa realizar inserção de dados. O propósito é similar à função scanf, da biblioteca padrão da linguagem C, mas é importante lembrar que o componente de campo de texto não é capaz de formatar dados, diferentemente da função scanf.

O botão consiste em uma caixa contendo texto que pode ser clicada pelo usuário. Na API, cada botão possui um identificador, especificado pelo programador, que serve para informar qual botão foi clicado em um evento de clique de botão (ver seção “Eventos”). Diferentemente do campo de texto, o botão não serve, geralmente, para informar dados, mas sim para solicitar que o programa realize uma ação. Em certo sentido, o botão é similar a pressionar a tecla “Enter”, ou ainda similar a escolher uma opção por meio de um caractere específico dentre uma lista de opções, em uma interface baseada em texto.

O retângulo consiste em caixa preenchida por uma cor sólida, que não tem interação com o usuário. O retângulo é uma forma geométrica bastante verstátil, portanto pode ser utilizado em diversos contextos, mas nesta API um uso possível é servir como plano de fundo para um texto estático. Basta criar um retângulo e após texto estático em algum lugar que o sobreponha, e haverá essencialmente texto estático com um plano de fundo.

A linha consiste em uma linha de 1 pixel de largura traçada de qualquer ponto da janela até qualquer outro ponto da janela. Por também se tratar de uma forma geométrica versátil, pode ser utilizada de diversas formas, mas um uso possível na API é servir como separador de seções ou de conteúdo.

Dicas

A cor é definida pela macro rgb(r, g, b). RGB é um esquema de cores que descreve um total de 2^24 cores por meio de combinações de três números de 0 a 255, sendo o primeiro a quantidade de vermelho, o segundo a quantidade de verde, e o terceiro a quantidade de azul. Naturalmente, existem ferramentas para escolher cores e obter seus valores no esquema RGB. Para isso, basta pesquisar em um motor de busca por “color picker”.

Diferentemente das funções disponíveis para programas baseados em texto, a API não possui em si a possibilidade de formatar dados, seja em leitura ou em escrita. Contudo, é possível utilizar as funções sprintf e sscanf, que funcionam como as funções printf e scanf, mas recebem ponteiros para strings para usar como base para a operação, ao invés dos arquivos padrões de entrada e saída. Por meio dessas funções é possível, por exemplo, ler um número em formato de string obtido de um campo de texto e transformá-lo em um inteiro ou valor de ponto flutuante utilizando a função sscanf.

A API não possui um índice Z explícito, ou seja, não há como explicitamente especificar qual componente deve estar na frente de qual. Contudo, a API trata as janelas que foram criadas primeiro como as que devem estar mais atrás, e as que foram criadas depois como as que devem estar mais à frente.

Funções

Janela

createWindow

Protótipo

Window createWindow(char *title, unsigned int x, unsigned int y, unsigned int width, unsigned int height, unsigned long color)

Parâmetros

  • char *title: String terminada em 0 que será o título exibido na janela
  • unsigned int x: Inteiro sem sinal que representa a distância da janela em relação à esquerda da tela, em pixels
  • unsigned int y: Inteiro sem sinal que representa a distância da janela em relação ao topo da tela, em pixels
  • unsigned int width: Inteiro sem sinal que representa a largura da janela, em pixels
  • unsigned int height: Inteiro sem sinal que representa a altura da janela, em pixels
  • unsigned long color: Inteiro sem sinal que representa a cor de fundo da janela. Veja a seção de dicas para como especificar cores

Retorno

Essa função retorna, de forma abstrata, a janela criada. A janela é essencialmente um ponteiro para uma estrutura utilizada internamente pela API. Essa referência para a janela é usada em quase todas as outras funções da API.

setWindowTitle

Protótipo

void setWindowTitle(Window window, char *title)

Parâmetros

  • Window window: A janela cujo título será alterado
  • char *title: String terminada em 0 que será o novo título exibido na janela

destroyWindow

Protótipo

void destroyWindow(Window window)

Parâmetros

  • Window window: A janela a ser destruída

Observação

Essa função irá destruir todos os componentes contidos na janela especificada antes de destruí-la. A função irá liberar a memória usada pela janela, mas é possível encerrar a aplicação sem utilizá-la.

Texto Estático

createStaticText

Protótipo

StaticText createStaticText(Window window, unsigned int x, unsigned int y, unsigned int width, unsigned int height, unsigned long color, char *text)

Parâmetros

  • Window window: janela na qual o componente será colocado
  • unsigned int x: Inteiro sem sinal que representa a distância do componente em relação à esquerda da janela, em pixels
  • unsigned int y: Inteiro sem sinal que representa a distância do componente em relação ao topo da janela, em pixels
  • unsigned int width: Inteiro sem sinal que representa a largura do componente, em pixels
  • unsigned int height: Inteiro sem sinal que representa a altura do componente, em pixels
  • unsigned long color: Inteiro sem sinal que representa a cor do texto. Veja a seção de dicas para como especificar cores
  • char *text: String terminada em 0 indicando o texto a ser exibido no componente

Retorno

Essa função retorna, de forma abstrata, o componente criado. O componente é essencialmente um ponteiro para uma estrutura utilizada internamente pela API. Essa referência para o componente é usada em quase todas as outras funções desse componente.
